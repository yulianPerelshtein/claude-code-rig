# Scope mismatch: ask, don't assume

When told "get X through" / "finish X", do not assume X is whatever you're
currently working on.

Clarify first:

- Is X **foundational work** that's already done and just needs merging?
- Or is X **new work** to build now?

These imply completely different actions (a merge/extract vs an implementation
phase). One clarifying question prevents building the wrong thing or extracting
a branch that's already shipped.
