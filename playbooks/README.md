# playbooks/

On-demand, scoped workflows. Pull via `@playbooks/<area>/<name>.md` rather than
auto-loading. A playbook is the **how-to**; the matching `domains/` doc (when
present) is the **fact/rule**. Indexed from `learnings/distilled.md`.

Areas: `wsl/`, `investigation/`, `refactoring/`, `debugging/`, `code-review/`,
`security/`, `ai-assisted-coding/`, `data-engineering/`, `skill-techniques/`,
`continuous-improvement/`, `prompt-optimization/`, `observability/`,
`token-economy/`, `ci/`.
